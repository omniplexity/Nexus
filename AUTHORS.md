## Contributors

Thank you to all contributors who have helped build Nexus. This project follows the [All Contributors](https://allcontributors.org) specification.

### How to Contribute

We welcome contributions from the community! See our [Contributing Guide](CONTRIBUTING.md) for details.

### Contribution Types

| Type | Description |
|------|-------------|
| 💻 Code | Bug fixes, features, refactoring |
| 📖 Docs | Documentation improvements |
| 🎨 Design | UI/UX improvements |
| 🧪 Testing | Test coverage and quality |
| 🔧 Maintenance | Dependencies, tooling |
| 💬 Community | Discord, forums, support |

### Contributors Guide

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/your-feature`)
3. **Make** your changes following our [Style Guide](STYLEGUIDE.md)
4. **Test** your changes (`npm run test`)
5. **Lint** your code (`npm run lint`)
6. **Format** your code (`npm run format`)
7. **Commit** using [Conventional Commits](https://www.conventionalcommits.org)
8. **Push** to your fork
9. **Open** a Pull Request

### Pull Request Guidelines

- PRs should be focused and atomic
- Include tests for new functionality
- Update documentation for any changes
- Follow the commit message convention
- Address review feedback promptly
