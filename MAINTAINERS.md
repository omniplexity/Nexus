# Maintainers

This document lists the project maintainers and their responsibilities.

## Core Maintainers

| Name | Role | Focus Areas |
|------|------|--------------|
| Nexus Team | Lead Maintainer | Architecture, Core Contracts, Orchestration |

## Roles and Responsibilities

### Lead Maintainer

- Final decision-making authority on architectural changes
- Setting technical direction and priorities
- Managing releases and versioning
- Resolving conflicts in pull requests
- Ensuring code quality and project standards

### Maintainer

- Reviewing and approving pull requests
- Triaging issues and bugs
- Writing and maintaining documentation
- Mentoring contributors
- Ensuring backward compatibility

## Becoming a Maintainer

The project may add new maintainers based on:

- **Contributions** — Consistent, high-quality contributions to the codebase
- **Community involvement** — Helping users, reviewing PRs, participating in discussions
- **Technical expertise** — Deep understanding of the project's architecture
- **Time commitment** — Availability to dedicate time to project maintenance

### Process

1. Current maintainers identify a candidate
2. Discussion among core maintainers
3. Invitation to maintainer role
4. Public announcement in the community

## Communication

- **Issues**: GitHub Issues for bug reports and feature requests
- **Discussions**: GitHub Discussions for Q&A and ideas
- **Security**: Direct email to security@nexus-ai.dev

## Decision Making

Technical decisions follow these principles:

1. **Consensus** — Seek agreement from maintainers when possible
2. **Documentation** — Document decisions in ADRs (Architecture Decision Records)
3. **Backward Compatibility** — Avoid breaking changes without clear migration path
4. **Transparency** — Make decisions and reasoning visible to the community

## Commit Access

Maintainers have:

- Write access to the main repository
- Ability to merge pull requests
- Access to release tooling

## Code of Conduct

All maintainers agree to follow the project's [Code of Conduct](CODE_OF_CONDUCT.md).
